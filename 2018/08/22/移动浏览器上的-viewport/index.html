移动浏览器上的 viewport

发表于 2018-08-22 | 分类于 前端 ， CSS |

在阅读本文之前，先需要了解 设备像素、设备独立像素、CSS-像素 和 桌面浏览器上的 viewport。

相信写过移动网站页面的同学，都见过上面这句话，只要加上这句话，我们的网页便可以适配不同尺寸不同分辨率的移动设备。那么这句话的原理是什么呢？我们下面便详细介绍一下移动浏览器上 viewport 的作用。

移动浏览器上有三个 viewport，分别是 layout viewport，visual viewport，ideal viewport，每个 viewport 代表着不同的含义。不同的移动设备上，这三个 viewport 的尺寸也都不相同。我们一般只关心 viewport 的宽度，而不太关心 viewport 的高度。

layout viewport

layout viewport 是网页布局的区域，是 <html> 元素的父容器，对应着桌面浏览器的 viewport。document.documentElement.clientWidth 用来表示 layout viewport 水平方向能够展示的 CSS 像素。在手机上浏览宽大的网页 (这些网页没有采用响应式设计) 的时候，layout viewport 通常超出了手机屏幕的范围，你只能一次浏览网页的一个部分，然后通过手指滑动浏览其他部分。

visual viewport

visual viewport 就是通过手机屏幕能够看到的网页区域。

ideal viewport

ideal viewport 不是某个区域，它是 设备独立像素。

画布和窗户

layout viewport 就像一块画布，网页上的元素就像画布上的画。visual viewport 就像一个窗户。我们站在窗户的一侧，通过窗户观看另一侧的画布上的画。画布的大小可以调整，当画布变大的时候，我们观察到的画的范围变小了(原来能看到画中的一张人脸，画布放大之后，只能看到一个眼睛了，但是这个眼睛的尺寸比原来大了)，当画布变小的时候，我们观察到的画的范围变多了。

layout viewport 有自己的物理尺寸，也就是在物理空间上占据的长度单位，单位是厘米、英寸等。layout viewport 还有自己的逻辑尺寸，也就是 CSS 的像素，单位是 px。逻辑尺寸和物理尺寸是没有关联的，当我们通过缩放来改变 layout viewport 的物理尺寸时候，layout viewport 的 CSS 像素是不会变化的。CSS 宽度为 980px 的 layout viewport，不管怎么缩放，它的 CSS 宽度永远是 980px。我们平时很少关心 layout viewport 的物理尺寸，更多时候只需要关心 layout viewport 的 CSS 像素。由于 layout viewport 是 <html> 元素的父容器，<html> 元素的默认宽度为 100%，所以 layout viewport 的 CSS 宽度，限定了网页中所有元素的 CSS 的宽度。

可以通过手动缩放来改变 layout viewport 的物理尺寸。当 layout viewport 物理尺寸变大的时候，就相当于画布变大了，虽然 layout viewport 的 CSS 像素不会发生变化，但是通过 visual viewport 能够观察到的 CSS 像素发生了变化。layout viewport 的物理尺寸越大，visual viewport 观察到的的 CSS 像素越少，相当于画布尺寸越大，我们通过窗户能够看到的画布的范围就越小。同理，layout viewport 的物理尺寸越小，visual viewport 的 CSS 像素越大。

visual viewport 的宽度

和 layout viewport 不一样，我们无法直接设置 visual viewport 的宽度，但是我们可以通过 <meta name="viewport"> 标签设置 layout viewport 的宽度，来间接影响 visual viewport 的宽度。

如果没有 <meta name="viewport"> 标签，那么 layout viewport 取默认值，在 iphone6 上是 980px，这时的 visual viewport 也是 980px，也就是说，整张网页的宽度正好占满手机屏幕的宽度。

如果为 <meta name="viewport"> 标签设置了 width 属性，但是没有设置 initial-scale 属性，这时 visual viewport 会尽量显示整张网页的内容，也就是说整张网页的宽度正好占满手机屏幕的宽度，因此不会显示滚动条。这里的整张网页的宽度，并不一定等于 layout viewport 的宽度，有可能大于 layout viewport。假如我们设置 <meta name="viewport" content="width=1000">，layout viewport 的宽度为 1000px，同时我们设置网页中的某个元素 width: 1500px;，那么 layout viiewport 的宽度并不会扩张，但是子元素会溢出，这时整张网页的宽度就是 1500px，visual viewport 的宽度也就是 1500px 了。但是 visual viewport 也不能无限扩大，它最大为手机屏幕物理像素的 2 倍。iphone6 的物理分辨率为 750 像素，因此 iphone6 的 visual viewport 最大为 1500px。假如我们设置某个元素为 width: 5000px;，或者设置 <meta name="viewport" content="width=5000">，在 iphone6 上 visual viewport 为 1500px，网页中超出 1500px 的部分会显示滚动条。

如果为 <meta name="viewport"> 标签设置了 initial-scale 属性，不管是否设置了 width 属性，visual viewport 的值都与 layout viewport 的值相同，也就是说整个 layout viewport 会占满手机屏幕。注意 layout viewport 并不一定等于整张网页的宽度。网页的宽度可以大于 layout viewport (通过设置某个子元素的 width 大于 layout viewport)，网页的宽度也可以小于 layout viewport (通过设置 <html> 元素的 width 小于 layout viewport)。但是如果不做特殊设置，网页的宽度默认与 layout viewport 的宽度是相等的 (<html> 元素的 width 默认为 100%)。
